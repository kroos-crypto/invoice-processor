📦 Rechnungsverarbeitung

Einrichtungsanleitung – Schritt für Schritt

1

Python prüfen (einmalig)

Öffne das Terminal (Mac: Cmd + Leertaste → „Terminal") und tippe:

python3 --version

Es muss 3.10 oder neuer erscheinen, z.B. Python 3.12.4

⚠ Version zu alt oder nicht gefunden? Lade Python kostenlos herunter: https://www.python.org/downloads/
→ Button „Download Python 3.x.x" klicken → Installieren → Terminal neu starten
2

App-Ordner öffnen & Pakete installieren

  1. Öffne das Terminal und wechsle in den App-Ordner (den Ordner, den du von mir erhalten hast):
    cd /Pfad/zu/invoice-processor # Beispiel Mac: cd ~/Downloads/invoice-processor # Beispiel Windows: cd C:\Users\Dein-Name\Downloads\invoice-processor
    💡 Tipp (Mac/Windows): Schreibe cd ins Terminal (mit Leerzeichen), dann ziehe den Ordner direkt ins Terminal-Fenster – der Pfad wird automatisch eingetragen.
  2. Installiere alle benötigten Pakete mit einem Befehl:
    pip3 install -r requirements.txt
    Das dauert 1–2 Minuten. Am Ende erscheint: Successfully installed ...
    ⚠ Fehler „pip3 not found"? Probiere stattdessen: pip install -r requirements.txt
3

Google Sheet erstellen

  1. Gehe zu https://sheets.google.com und erstelle ein neues, leeres Sheet.
  2. Gib dem Sheet einen Namen, z.B. „Transportkosten 2025".
  3. Notiere die Spreadsheet-ID aus der Browser-URL:
    https://docs.google.com/spreadsheets/d/HIER_IST_DIE_ID/edit # Die ID ist der lange Code zwischen /d/ und /edit # Beispiel: 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms
    Kopiere diese ID – du brauchst sie in Schritt 5.
4

Google Service Account erstellen (einmalig)

Der Service Account ist ein „Roboter-Nutzer", der im Hintergrund Daten ins Sheet schreibt. Kostenlos.

  1. Öffne die Google Cloud Console:
    https://console.cloud.google.com
    Mit deinem Google-Konto einloggen.
  2. Klicke oben links auf das Projekt-Dropdown → „Neues Projekt".
    Name: z.B. rechnungen → Erstellen.
  3. Klicke links im Menü auf „APIs und Dienste" → „Bibliothek".
    Suche nach Google Sheets API → Klicke drauf → „Aktivieren".
    Dann zurück zur Bibliothek → suche Google Drive API → „Aktivieren".
  4. Klicke links auf „Anmeldedaten" → oben auf „+ Anmeldedaten erstellen" → „Dienstkonto".
  5. Fülle das Formular aus:
    FeldWas du einträgst
    Name des Dienstkontosrechnungen-bot (beliebig)
    Dienstkonto-IDwird automatisch ausgefüllt
    Beschreibungleer lassen
    → Erstellen und fortfahren → Weiter → Fertig.
  6. Du siehst jetzt das neue Dienstkonto in der Liste. Klicke auf die E-Mail-Adresse (sieht aus wie rechnungen-bot@rechnungen-xxxxx.iam.gserviceaccount.com).
  7. Gehe auf den Tab „Schlüssel" → „Schlüssel hinzufügen" → „Neuen Schlüssel erstellen" → JSON → Erstellen.
    📥 Eine .json Datei wird heruntergeladen. Das ist dein Zugriffsschlüssel. Bewahre ihn sicher auf – nicht teilen!
  8. Benenne die Datei um in credentials.json und lege sie in den invoice-processor Ordner.
5

Service Account mit dem Sheet verbinden

  1. Öffne die credentials.json in einem Texteditor (z.B. TextEdit, Notepad).
    Suche den Wert bei "client_email" – das ist die E-Mail-Adresse des Service Accounts.
    Beispiel: rechnungen-bot@rechnungen-123456.iam.gserviceaccount.com
  2. Öffne dein Google Sheet (aus Schritt 3).
    Klicke oben rechts auf „Teilen" (grüner Button).
  3. Füge die E-Mail-Adresse des Service Accounts ein → Berechtigung: „Bearbeiter" → Senden.
    💡 Jetzt kann die App automatisch ins Sheet schreiben.
6

Konfigurationsdatei anlegen

  1. Öffne den invoice-processor Ordner.
    Du siehst eine Datei namens .env.example – kopiere sie und benenne die Kopie um zu .env.
    ⚠ Mac-Tipp: Dateien die mit . beginnen sind versteckt. Drücke in Finder Cmd + Shift + . um sie sichtbar zu machen.
  2. Öffne .env im Texteditor. Es steht drin:
    GOOGLE_CREDENTIALS=credentials.json SPREADSHEET_ID=your_spreadsheet_id_here
    Ersetze your_spreadsheet_id_here mit der ID aus Schritt 3. Speichern.
    💡 Fertige .env Datei sieht so aus:
    GOOGLE_CREDENTIALS=credentials.json SPREADSHEET_ID=1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms
7

App starten

  1. Terminal öffnen → in den App-Ordner wechseln (wie in Schritt 2):
    cd /Pfad/zu/invoice-processor
  2. Umgebungsvariablen laden und App starten:
    # Mac / Linux: export $(cat .env | xargs) && python3 app.py # Windows (PowerShell): Get-Content .env | ForEach-Object { $name, $value = $_ -split '=', 2; [Environment]::SetEnvironmentVariable($name, $value) }; python app.py
  3. Wenn alles funktioniert siehst du:
    * Running on http://0.0.0.0:5000 * Debug mode: on
  4. Öffne deinen Browser und gehe zu:
    http://localhost:5000
    ✅ Die App läuft! Du siehst jetzt die Drag & Drop Oberfläche.
⚠ App beenden: Im Terminal Ctrl + C drücken.
⚠ Beim nächsten Mal: Einfach wieder Schritt 7 ausführen – Schritte 1–6 sind einmalig.
8

App benutzen

  1. Kategorie wählen – immer als erstes auswählen:
    Transport | Zoll | Lagerkosten & Diverse
  2. Referenz eintragen (nur wenn nötig):
    Wenn die Rechnung keine OR.../BL.../PBL... Nummer enthält (z.B. Expeditors, Lagerkosten) → hier manuell eintragen.
  3. Dateien hochladen – PDF, CSV oder XLSX per Drag & Drop ins Feld ziehen
    (oder klicken → Datei auswählen). Mehrere Dateien gleichzeitig möglich.
  4. „Rechnungen verarbeiten" klicken → die App erkennt automatisch den Anbieter,
    liest alle Felder aus und schreibt die Daten ins Google Sheet.
  5. Duplikat-Warnung: Wenn du eine Rechnung ein zweites Mal hochlädst, fragt die App nach:
    → Überspringen (Rechnung ignorieren) oder Trotzdem importieren (doppelt eintragen)
ℹ Google Sheet Tabs
Die App erstellt automatisch 3 Tabs im Sheet:
  • Transport – FedEx, UPS, Transdirekt, Raben, Logfret
  • Zoll – FedEx Zoll, UPS Zoll, Expeditors
  • Lagerkosten & Diverse – alle sonstigen Rechnungen

Häufige Probleme & Lösungen

FehlermeldungLösung
ModuleNotFoundError: flask Pakete nicht installiert → Schritt 2 wiederholen
FileNotFoundError: credentials.json Die credentials.json liegt nicht im App-Ordner → prüfen
SPREADSHEET_ID nicht konfiguriert .env Datei fehlt oder Schritt 7 ohne export $(cat .env | xargs) gestartet
gspread.exceptions.SpreadsheetNotFound Sheet-ID falsch oder Service Account hat keinen Zugriff → Schritt 5 wiederholen
Port 5000 bereits belegt Am Ende von app.py Port ändern: port=5001 → dann http://localhost:5001
Anbieter nicht erkannt Der Fallback-Parser übernimmt trotzdem. Referenz und Kategorie manuell prüfen.

🎉 Bereit!

Einrichtung abgeschlossen. Die App läuft auf http://localhost:5000

Bei Problemen: Fehlermeldung aus dem Terminal kopieren und melden.
